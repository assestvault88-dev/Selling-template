üîß Troubleshooting Guide

Comprehensive solutions for common issues you might encounter during development.

‚ö†Ô∏è First Step: Always check your browser's Developer Console (F12 or Right Click > Inspect > Console) for red error messages.

Visual & Layout Issues

‚ùå CSS Variables Not Updating

Cause: Browser caching old CSS file or syntax error in variable declaration.

Solution:

  1. Hard refresh the page (Ctrl + F5 or Cmd + Shift + R)
  2. Check if you missed a semicolon ; or closing brace } in :root
  3. Verify you are editing styles/main.css and not a minified version

‚ùå Mobile Menu / Sidebar Not Toggling

Cause: JavaScript event listeners not attached because script loaded too early.

Solution:

  1. Ensure <script src="scripts/app.js"> is at the very bottom of the <body> tag
  2. Or wrap your initialization code in:
    document.addEventListener('DOMContentLoaded', () => { // your init code });

‚ùå Images Broken or Not Loading

Cause: Incorrect relative paths, especially when moving files between folders.

Solution:

  • If HTML is in root: src="images/logo.png"
  • If HTML is in subfolder: src="../images/logo.png"
  • Space in filename: Avoid spaces! Use my-image.png instead of my image.png

Charts & Data Issues

‚ùå "Chart is not defined" Error

Cause: The Chart.js library script is missing or loaded after your app code.

Solution: Check your head or body tag order:

<script src="https://cdn.jsdelivr.net/npm/chart.js"></script> <script src="scripts/app.js"></script>

‚ùå Charts Look Blurry or Pixelated

Cause: High DPI (Retina) displays scaling issues.

Solution: Add options.devicePixelRatio to your chart config or ensure responsive: true is set (default).

‚ùå Canvas is Already in Use

Cause: Re-initializing a chart on the same canvas ID without destroying the old one.

Solution:

let myChart; // ... inside your update function: if (myChart) { myChart.destroy(); // Destroy old instance } myChart = new Chart(ctx, config); // Create new

Network & API Issues

‚ùå "Cross-Origin Request Blocked" (CORS)

Cause: Security feature attempting to load local files via file:// or API blocking your origin.

Solution:

  1. Never use file:// protocol for APIs. Use a local server (see Getting Started).
  2. If accessing external API, ensure the server supports CORS or use a proxy.

‚ùå 404 Not Found on Deployment

Cause: 1. Case sensitivity (Linux servers treat Icon.png and icon.png differently).
2. Missing .html extension in links if server doesn't auto-resolve.

Solution:

  • Make all filenames lowercase (e.g., profile.html, not Profile.html).
  • Check references: href="about.html" vs href="/about".

Performance Warning

‚ö†Ô∏è Page Lagging / High Memory

Cause: Creating too many chart instances or not cleaning up event listeners.

Solution: Audit your loops. Do not create new Chart instances inside a loop without destroying old ones.

üé® Customization Guide

Changing Colors

Open styles/main.css and modify the CSS variables:

:root {
--color-bg: #000000; /* Background */
--color-accent: #87CEEB; /* Accent (sky blue) */
--color-success: #4caf50; /* Positive values */
--color-danger: #f44336; /* Negative values */
}

Changing Logo

Find and replace in all HTML files:

<div class="logo">Financila</div>

<div class="logo">YourBrand</div>

Adding New Charts

In scripts/app.js, add to the viewData object:

newChart: {
title: 'My New Chart',
type: 'bar',
data: {
labels: ['A', 'B', 'C'],
datasets: [{
label: 'Dataset',
data: [10, 20, 30],
backgroundColor: '#87CEEB'
}]
}
}

üîå API Integration

Fetching Real Data

Replace mock data with API calls:

async function loadData() {
try {
const response = await fetch('https://your-api.com/data');
const data = await response.json();

// Update chart
viewData.overview.charts.revenue.data.datasets[0].data = data.revenue;
renderChart('revenue');

// Update KPI cards
document.getElementById('total-revenue').textContent = '$' + data.totalRevenue;
} catch (error) {
console.error('Error loading data:', error);
}
}

document.addEventListener('DOMContentLoaded', loadData);
